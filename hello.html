ORM: Object Relational Mapping

RDB 테이블을 객체지향적으로 사용하기 위한 기술 관계형DB를 객체지향과 매핑시킨다는 추상화된 개념 JPA는 ORM 개념을 java 언어에서 구현하기 위한 표준 기술 ORM을 프로젝트에 도입하면 자동 생성되는 SQL문들의 성능 개선을 위한 튜닝 작업이 어려움 쿼리를 작성하지 않지만, 내부적으로는 쿼리가 만들어지고 실행됨

JPA Provider: JPA 구현체

그것이 Hibernate. 오픈소스 (Hibernate, EclipseLink, DataNucleus)

JPA 장점

객체지향적으로 데이터를 관리하며 비지니스 로직에 집중 가능 테이블 구조를 객체로 관리(Model) 기본 Entity에 대해 CRUD 자동 생성 사용자 정의 네임스페이스 가능 데이터를 캐시해두는 저장소(Repository)관리를 효율적으로 하도록 지원

JPA 단점

객체지향 및 Spring MVC패턴을 정확히 이해해야 함 공부할게 많고 특정 데이터베이스의 강력함을 활용할 수 없다. (통계, 데이터 집계, 배치...)
